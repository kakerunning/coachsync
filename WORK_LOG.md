**What was done:**
Three separate errors were diagnosed and resolved. (1) Turbopack junction point panic on startup was caused by a stale `.next` cache on Windows — fix is `rmdir /s /q .next`. (2) "fetch failed" on signup and "PGlite failed to initialize" were both caused by Prisma's embedded local database (`prisma+postgres://`) not being supported on Windows. The project was migrated to Supabase by updating `DATABASE_URL` in `.env` and removing the now-invalid `url` field from `schema.prisma` (Prisma v7 reads the URL from `prisma.config.ts` only). `prisma generate` completed successfully. (3) `prisma migrate deploy` returned P1001 (unreachable) because the command was executed from WSL2, which does not share Windows network routes to Supabase — it must be run directly from a Windows PowerShell or CMD terminal.
