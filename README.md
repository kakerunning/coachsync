# CoachSync

**A full-stack coach-athlete management platform with multilingual support and data visualization.**

CoachSync helps athletics coaches and their athletes communicate across language barriers — managing training sessions, tracking personal records, visualizing performance trends, and generating PDF reports, all in one place.

## Features

### For Coaches
- **Athlete roster management** — invite athletes via shareable link, view all linked athletes
- **Session logging** — create training sessions with warmup sets, laps, and distance data
- **Event scheduling** — schedule competitions, camps, and time trials
- **Feedback & comments** — annotate athlete sessions with coaching notes
- **Multilingual comments** — auto-translate notes via DeepL API (supports EN/DE/JA and more)
- **Analytics dashboard** — interactive Plotly charts for volume trends, fatigue timelines, performance curves
- **PDF weekly reports** — downloadable PDF summaries generated by Python (fpdf2)

### For Athletes
- **Personal dashboard** — greeting, weekly volume, training streak, upcoming events
- **Personal records tracking** — log and view PBs by discipline
- **Calendar view** — month/week/day view of sessions and events
- **Push notifications** — PWA-based web push for session reminders (VAPID)
- **Session history** — view all training with set/lap breakdowns and RPE feedback

### Architecture Highlights
- **Dual-service design** — Next.js handles auth and DB; a separate Python FastAPI service handles all computation-heavy chart and PDF generation
- **Role-based access control** — `COACH` and `ATHLETE` roles with per-route enforcement at both API and UI layers
- **Internal service auth** — Python service protected by `X-Internal-Key` header; not exposed to the internet
- **Multilingual-first** — per-user language preference; DeepL translation integrated at the comment level

## Getting Started

### Prerequisites

- Node.js 20+
- Python 3.12+
- A [Supabase](https://supabase.com) project (free tier works)
- A [DeepL API](https://www.deepl.com/pro-api) key (free tier works)

### 1. Clone and install

```bash
git clone https://github.com/your-username/coachsync.git
cd coachsync/coachsync
npm install
```

### 2. Configure environment variables

Copy `.env.example` to `.env.local` and fill in your values:

```bash
cp .env.example .env.local
```

```env
DATABASE_URL="postgresql://..."
DIRECT_URL="postgresql://..."
AUTH_SECRET="run: openssl rand -hex 32"
NEXTAUTH_URL="http://localhost:3000"
DEEPL_API_KEY="your-deepl-key"
VIZ_SERVICE_URL="http://localhost:8000"
VIZ_INTERNAL_KEY="run: openssl rand -hex 32"
VAPID_PUBLIC_KEY="..."
VAPID_PRIVATE_KEY="..."
VAPID_MAILTO="mailto:you@example.com"
```

### 3. Set up the database

```bash
npx prisma generate
npx prisma migrate deploy

# Optional: seed with demo data (coach + 4 multilingual athletes)
npx prisma db seed
```

### 4. Start the Python visualization service

```bash
cd ../viz
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### 5. Start the Next.js app

```bash
cd ../coachsync
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

```
    ├── pdf/                     # PDF report generation
    └── routers/                 # FastAPI route handlers
```

### Backend Pattern

Each domain follows a three-layer pattern:

```
API Route  →  Service (business logic)  →  Repository (DB queries)
```

Services return typed discriminated unions — `{ ok: true, data }` or `{ ok: false, status, error }` — so API routes never touch Prisma directly and errors are explicit.

## API Overview

| Method | Route | Description |
|--------|-------|-------------|
| `POST` | `/api/auth/register` | Create account (COACH or ATHLETE) |
| `GET/POST` | `/api/sessions` | List / create training sessions |
| `GET/PATCH/DELETE` | `/api/sessions/[id]` | Session CRUD |
| `GET/POST` | `/api/events` | List / create events |
| `GET/POST/DELETE` | `/api/athletes` | Coach roster management |
| `GET/POST` | `/api/records` | Personal records |
| `GET` | `/api/calendar` | Calendar items (sessions + events) |
| `POST` | `/api/viz/charts` | Generate Plotly chart (proxied to Python) |
| `POST` | `/api/viz/reports` | Generate PDF report (proxied to Python) |
| `POST` | `/api/push/subscribe` | Register PWA push subscription |
| `POST` | `/api/invite/[token]` | Accept coach invite link |

## Key Engineering Decisions

**Why a separate Python service for charts?**
Pandas and Plotly are significantly more capable than JS equivalents for statistical chart generation. Keeping the compute-heavy work in Python also keeps the Next.js server lean and lets each service scale independently.

**Why fpdf2 for PDFs?**
No GTK/wkhtmltopdf dependency means the service runs cleanly in Docker on any OS (including Windows) without system-level installs.

**Why Prisma v7 with `PrismaPg` adapter?**
Direct adapter usage (passing `{ connectionString }` instead of a `Pool` instance) avoids the TypeScript type conflict between `pg` and Prisma's bundled types while supporting PgBouncer connection pooling for Supabase.

**Hydration-safe date rendering**
Server and client clocks diverge on `new Date()`. All "today" values use `useState("") + useEffect` so the server renders a neutral placeholder and the client fills in the real date after mount — no React hydration warnings.
